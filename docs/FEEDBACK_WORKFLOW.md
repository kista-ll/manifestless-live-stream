# Codex指摘反映ワークフロー

## 指摘を恒久ルールにする命令

```text
この指摘を再発防止ルールとして扱ってください。
docs/CODEX_RULES.mdの登録基準を確認し、必要ならActiveルールとして追記してください。
既存ルールまたはADRと重複する場合は新規追加せず、該当箇所を更新してください。
ルール追加に伴って必要なテスト、テンプレート、ドキュメントも同じ変更内で更新してください。
最後に、追加・更新したルールIDと検証方法を報告してください。
```

単発修正に限定する場合:

```text
今回は単発修正です。docs/CODEX_RULES.mdには追加しないでください。
```

## Codexの処理手順

1. 指摘を再利用可能な一般ルールへ変換する。
2. 既存ルールを検索する。
3. 重複があれば既存ルールを更新する。
4. 設計判断なら`docs/DECISIONS.md`へADRを追加する。
5. 実装ルールなら`docs/CODEX_RULES.md`へ追加する。
6. 必要なテストや静的チェックを追加する。
7. 検証後、ルールID・変更箇所・結果を報告する。

## 追加対象

- 複数タスクで繰り返し適用するルール
- 同じ指摘を再度受ける可能性があるルール
- セキュリティ、品質、運用上の禁止事項
- レビュー時に確認可能なルール

## 追加しないもの

- 誤字修正
- 特定ファイルだけの一時対応
- 既存ルールと重複する内容
- 実験中だけの調査メモ
